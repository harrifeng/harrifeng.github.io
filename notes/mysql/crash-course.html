Chapter 01: Understanding SQL

Database Basics

What Is a Database?

  • 我们要分清两个概念:
    • Oracle, Mysql这种软件叫做"数据库管理系统"(DBMS)
    • 数据库是存放某些数据的容器,比如ct_development就是一个数据库

Tables

  • Table是存储"某一种类型"数据的结构体 
    Table: A structured list of data of a specific type.
  • table名字在某个database里面唯一即可.
  • Schema是描述table的layout(table里面存什么数据, 每一行叫什么等等)或者数据库本身的信息 
    Schema: Information about database and table layout and properties.

Columns and Datatypes

  • Column 就是table里面的"一种信息", 每个table会有"多种信息" 
    Column: A single field in a table. All tables are made up of one or more columns.
  • 每个column都有自己的类型,就好比程序里面的变量一样.类型都不一定相同 
    Datatype: A type of allowed data. Every table column has an associated
datatype that restricts(or allows) specific data in that column

Rows

  • Row就是table里面的"一个单位" "一个记录" "一个完整的信息". 而table是这么"多个信息"的集合. 
    Row: A record in a table

Primary Keys

  • 每一个table都必须依靠某个column(或者多个column)来唯一标示自己. 
    Primary Key: A colummn (or set of columns) whose values uniquely identify every row in a table
  • 成为primary key的要求也不多:
    • 对这个column来说, 没有两个记录(row)是一样的
    • 对这个column来说, NULL是不可接受的值
  • 多个column都是primary key的情况也是可以的. 那么就是如下的要求:
    • 对于这几个column来说,他们的组合必须不能重复(其中某一个column重复可以)
    • 对于这几个column来说, NULL是不可接受的.
  • 对于primary key来说,除了上面的"硬性要求", 还有一些best practice:
    • 不要update primry key column里面的值
    • 不要重复利用primary key column里面的值
    • 不要使用"可能改变"的值作为primary key (比如你把name作为供应商table的 primary key就不合理, 因为供应商可能会改名或者重组!)

What Is SQL?

  • SQL是Structured Query Language的简称.
  • 虽然多数DBMS都会支持SQL,但很少有两种DBMS实现SQL的方式相同.

Chapter 02: Introducing MySQL

What Is MySQL?

  • MySQL是一种开源的DBMS, 功能不如传统数据库全面,而且经常加入新的特性

Client-Server Software

  • DBMS其实分成两类:
    • filebase: 比如Access但是关键场合不能使用
    • Client-Server: MySQL就是这类
  • 既然是Client-Server的,那么就是Server负责管理数据,client只是负责向Serer提问:
    • server 端是: MySQL DBMS
    • client 端是: MySQL-provided tool, 或是其他语言,比如Perl, Ruby, Java

MySQL Tools

  • mysql的server是在后台运行的,所以mysql也提供了一些client的工具让我们使用

mysql Command-Line Utility

  • 随着MySQL安装的bash命令 mysql就是一个简单的client端 
    $ mysql -uroot

Retrieving Distinct Rows

  • 我们取得一个column的时候, 如果不是primary key的话,内容是可以重复的.如果我们 不想看到这些重复的内容.我们需要DISTINCT关键字 
    mysql> SELECT company_id FROM members;
SELECT company_id FROM members;
+------------+
| company_id |
+------------+
|          1 |
|          1 |
|          1 |
|          1 |
|          1 |
|          1 |
|          1 |
|          2 |
|          2 |
|          3 |
|          3 |
+------------+
11 rows in set (0.00 sec)

mysql> SELECT DISTINCT company_id FROM members;
SELECT DISTINCT company_id FROM members;

| company_id |
+------------+
|          1 |

    mysql> SELECT DISTINCT company_id, admin_type FROM members;

Limiting Results

  • 如果结果太多,我们想只看一部分,那么就要用LIMIT:
    • LIMIT只有一个参数的话,就是从头开始多少个 
      mysql> SELECT email FROM members LIMIT 5;
SELECT email FROM members LIMIT 5;

| carlin@cubetree.com          |

      mysql> SELECT email FROM members LIMIT 0, 4;
