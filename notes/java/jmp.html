进程和多线程的概念及线程的优点

使用多进程

继承Thread类

  • 最简单的实现thread的方法是继承Thread类, 然后实现run()方法.其实run()方法是 在接口Runnable里面定义的,而Thead实现了Runnable接口 
    public class Thread implements Runnable
  • 因为main函数本身就是一个thread,所以,我们可以只创建一个thread:MyThread,然后 和main函数交替随机运行来展示下thread的用法:
    • MyThread会随机运行五次 
      package org.hfeng.jmp.c01.t03;

public class MyThread extends Thread {
    @Override
    public void run() {
        try {
            for (int i = 0; i < 5; i++) {
                int time = (int) (Math.random() * 1000);
                Thread.sleep(time);
                System.out.println("run=" + Thread.currentThread().getName());
            }
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}
    • main thread也会随机运行五次 
      package org.hfeng.jmp.c01.t03;

public class Test {
    public static void main(String[] args) {
        try {
            MyThread thread = new MyThread();
            thread.setName("myThread");
            thread.start();

            for (int i = 0; i < 5; i++) {
                int time = (int) (Math.random() * 1000);
                Thread.sleep(time);
                System.out.println("main=" + Thread.currentThread().getName());
            }
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

  • 从上面的例子我们可以看到,在MyThread里面我们是override了函数run(),但是在创建 好thread之后开始调用的时候,使用的是start()函数. 这是多线程调用的特点.Thread.java 类中的start()方法是用来通知"线程规划器"此线程已经准备就绪. 
    start() causes this thread to begin execution; the Java Virtual
Machine calls the run() method of this thread.
The result is that two threads are running concurrently: the current
thread(which returns form the call to the start method) and the other
thread(which executes its run method)
  • 如果你创建了线程实例之后,马上调用的不是start()而是run(),那就和普通的函数调 用没有区别了.调用结果就一定是 
    ////////////////////////////////////////////////////

// run=main                                       //
// run=main                                       //
// run=main                                       //
// run=main                                       //
// main=main                                      //
// main=main                                      //
// main=main                                      //
// main=main                                      //
// main=main                                      //
////////////////////////////////////////////////////
  • 还有需要了解的就是先start()的,并不一定先被JVM调度.

实现Runnable接口

  • 另外一个创建thread的方法是implements Runnable接口.因为java是单继承,如果类 已经了其他的类,那么想要把自己变成thread,就只能实现Runnable接口
  • 前面也讲到过,Thread.java其实就是implements了Runnable接口,那么为什么不"只有 一个Runnable接口"就完了呢,还要Thread.java. 原因就是前面说的start()通知"线程 规划器",然后"线程规划器"再去调用thread.这些逻辑必须在一个class,而不是一个 interface里面实现.所以我们就有了Thread.java.
  • 下面使用Runnable的例子你也会发现,即便是你使用了Runnable来创建thread.调用的 时候,还是会引出Thread:
    • MyRunnable: 
      package org.hfeng.jmp.c01.t04;

public class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println("In a runnable");
    }
}
    • Test: 
      package org.hfeng.jmp.c01.t04;

public class Test {
    public static void main(String[] args) {
        Runnable runnable = new MyRunnable();
        Thread thread = new Thread(runnable);
        thread.start();
        System.out.println("Thread done!");
    }
}
  • 我们看到,上例中的Thread(Runnable target)的ctor. 这一段其实如果不看源代码不 是很好理解.其意思:
    • 是使用一个runnable实例来"初始化一个新的Thread", 但是又有所不同,因为新创 建的thread object有内部的private变量指向runnable 
      public
class Thread implements Runnable {
    /* Make sure registerNatives is the first thing <clinit> does. */
    private static native void registerNatives();
    static {
        registerNatives();
    }

    private volatile char  name[];
    private int            priority;
    private Thread         threadQ;
    private long           eetop;

    /* Whether or not to single_step this thread. */
    private boolean     single_step;

    /* Whether or not the thread is a daemon thread. */
    private boolean     daemon = false;

    /* JVM state */
    private boolean     stillborn = false;

    /* What will be run. */
    private Runnable target;

    // .....
}
    • 这也就是说如果Thread(r1)连续调用两边.两个Thread虽然有不同的id,不同的名字, 但是其private runnable却都指向同一个runnable object.这有时候会带来一个问 题:线程已经是不同的线程,但是却共享了一个private object. 这个private object 内部的private 变量就被多个thread共享了!下一节会讲到这个例子.

实例变量与线程安全

  • thread自己的stack是独有的,不跟其他的thread分享的.所以,绝大多数情况下,thread 的private 成员变量是"不共享"的,也就是不会出现线程安全的问题:
    • MyThread 
      package org.hfeng.jmp.c01.t05;

public class MyThread extends Thread {
    private int count = 5;

    public MyThread(String name) {
        super();
        this.setName(name);
    }

    @Override
    public void run() {
        super.run();
        while (count > 0) {
            count--;
            System.out.println(this.currentThread().getName() + ", count = " + count);
        }
    }
}
    • Test 
      package org.hfeng.jmp.c01.t05;

public class Test {
    public static void main(String[] args) {
        MyThread a = new MyThread("A");
        MyThread b = new MyThread("B");
        MyThread c = new MyThread("C");
        a.start();
        b.start();
        c.start();
    }
}

  • 只所以说绝大多数情况,是因为在某些情况下,使用Thread(Runnable target)来初始化 thread,但是`target`使用了同一个,导致了Thread虽然有多个,但是其内部private变 量都是指向同一个`target` runnable instance.所以以另外一种方式"共享了private 变量"
  • thread共享的是global, static变量,但是这些都是c语言里面的概念,在Java里面想 实现这些例子不是那么的容易,我们需要设计特殊的thread类让他们从类的initializer 里面获取这些'global变量',也就是一个data类:
    • data类 
      package org.hfeng.jmp.c01.t07;

public class Data {
    private int count;

    public Data(int count) {
        this.count = count;
    }

    public void increase() {
        this.count++;
    }

    public void display() {
        System.out.println("Count is now: " + count);
    }
}
    • thread类 
      package org.hfeng.jmp.c01.t07;

public class MyThread extends Thread {
    private Data data;

    public MyThread(Data data) {
        this.data = data;
    }

    @Override
    public void run() {
        for (int i = 0; i < 1000; i++) {
            data.increase();
        }
        data.display();
    }
}
    • test类 
      package org.hfeng.jmp.c01.t07;

public class Test {
    public static void main(String[] args) {
        Data data = new Data(0);
        Thread a = new MyThread(data);
        Thread b = new MyThread(data);
        a.start();
        b.start();

        while(a.isAlive() || b.isAlive()) {
            ;
        }
        data.display();
    }
}

  • 原因很明显是shared variable没有synchronized, 所以最后没有增加到2000,改正方 法就是把增加synchronized关键字给increase函数,让大家按顺序来. 
    public synchronized void increase() {
    this.count++;
}
