活动的基本用法

  • 我们新建一个工程,但是这次选择, 没有默认activity的工程
  • 我们选择新建一个FirstActivity函数,继承于Activity 
    package com.hfeng.example.activitytest;

import android.app.Activity;
import android.os.Bundle;

public class FirstActivity extends Activity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
    }
}
  • 我们新建resouce下面的文件夹名字叫做layout,然后新建一个xml的布局文件first_layout 
    <?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:orientation="vertical" android:layout_width="match_parent"
    android:layout_height="match_parent">
    <Button
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:text="Button 1"
        android:id="@+id/button_1"
        />
</LinearLayout>
  • Layout就是图像的一部分,想要这个对象能够被用户所调用,我们要把"设计"在Activity 中"加载". 联系的方法如下

  • 我们看到,我们使用的是setContentView函数, 这个函数需要制定一个id
  • 我们id的获取方式比较特别是R.layout.first_layout. 这个R看起来应该是一个class 但是我们却没有去特别的include它,为什么呢? 因为它就在当前package下!这个R class 的内容如下

在AndroidManifest文件中注册

  • 一个我们刚才其实是把layout文件注册到activity里面去
  • 其实activity文件也要注册到AndroidManifest.xml里面去,如下 
    <manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.hfeng.example.activitytest">

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:theme="@style/AppTheme">
        <activity
            android:name=".FirstActivity"
            android:label="First Activity Label">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>
    </application>

</manifest>
  • 其中<activity></activity>是新加的部分:
    • android.name就是com.hfeng.example.FirstActivity,只不过用'.'来表示全称
    • android.label是标题
    • <intent-filter>里面的两句,是为了让启动的时候自动加载当前activity.
