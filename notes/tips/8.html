Chapter 8: Programming Practices

  • 很多语言都会有自己的局限性,而当这个语言被广泛利用的时候,局限性会更加突出的展 现出来
  • 自从Ajax出现以后,javascript被压榨的更加厉害.为了应对这种压榨,出现了一些best practice. 这些best proactice能够让js更好的在web端发挥作用

Avoid Double Dvaluation

  • 和其他语言一样,js允许你传入一段"字符串",这段"字符串"其实是一段代码,我们可以 运行这段代码,有如下四种方法可以完成这个功能:
    • eval() 
      var num1 = 5;
var num2 = 6;

// eval()
console.log(eval("num1 + num2"));

////////////////////////////////////////////////////
// <===================OUTPUT===================> //
// 11                                             //
////////////////////////////////////////////////////
    • Function() constructor 
      var num1 = 5;
var num2 = 6;

// Function()
sum = new Function("arg1", "arg2", "return arg1 + arg2");
console.log(sum(num1, num2));

////////////////////////////////////////////////////
// <===================OUTPUT===================> //
// 11                                             //
////////////////////////////////////////////////////
    • setTimeout() 
      <!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8">
    <title>Tester</title>
  </head>
  <body>

    <script>
     setTimeout("alert('Eval in setTimeout')", 100);
    </script>
  </body>
</html>
    • setInterval() 
      <!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8">
    <title>Tester</title>
  </head>
  <body>

    <script>
     setInterval("alert('Eval in setInterval')", 100);
    </script>
  </body>
</html>
  • 但是"使用字符串作为代码"的方式,却会让你付出"二次评估"的代价,根据不同的浏览器 运行如下两行代码会有巨大的时间消耗的不同 
    // faster
var item = array[0];

// slower
var item = eval_r("array[0]");

Use Object/Array Literals

  • 有很多种创建object和array的方式,但是在这些方法中,速度最快的,就是使用object literal和array literal.而且literal的方式会占用更少的代码空间 
    // create an object
var myObject = {
  name: "Nicholas",
  count: 50,
  flag: true,
  pointer: null
};

// create an array
var myArray = ["Nicholas", 50, true, null];

Don't Repeat Work

  • DRY(Don't Repeat Yourself)是常见的代码规范,在js里面遵循这个规范会同时带来代 码速度的提高
  • 最常见的代码重复工作,就是浏览器检测 
    function addHandler(target, eventType, handler){
  if (target.addEventListener){
    //DOM2 Events
    target.addEventListener(eventType, handler, false);
  } else { //IE
    target.attachEvent("on" + eventType, handler);
  }
}
function removeHandler(target, eventType, handler){
  if (target.removeEventListener){
    //DOM2 Events target.removeEventListener(eventType, handler, false);
  } else { //IE
    target.detachEvent("on" + eventType, handler);
  }
}
  • 在Jquery诞生以后,这种探测其实已经不再是普通js程序员的工作了,我们这里只是举 一个例子
  • 这种看起来很"精炼"的代码其实在背后有着隐藏的性能问题,因为每次函数调用都要执 行重复的"探测"工作.有多种方法能够避免这一点

Lazy Loading

  • 第一种避免重复的办法就是延迟加载,所谓延迟加载就是"直到用到某个变量"才去初 始化这个变量(其实就是把function重新赋值). 很炫的一个技巧 
    function addHandler(target, eventType, handler){
  //overwrite the existing function
  if (target.addEventListener){ //DOM2 Events
    addHandler = function(target, eventType, handler){
      target.addEventListener(eventType, handler, false); };
  } else { //IE
    addHandler = function(target, eventType, handler){
      target.attachEvent("on" + eventType, handler); };
  }
  //call the new function addHandler(target, eventType, handler);
}
function removeHandler(target, eventType, handler){
  //overwrite the existing function
  if (target.removeEventListener){ //DOM2 Events
    removeHandler = function(target, eventType, handler){
      target.addEventListener(eventType, handler, false); };
  } else { //IE
    removeHandler = function(target, eventType, handler){
      target.detachEvent("on" + eventType, handler); };
  }
  //call the new function removeHandler(target, eventType, handler);
}
  • 调用延迟加载的函数,在第一次的时候总会慢一点,但是后面的调用会"得益于"第一次 的调用,从而加快函数的速度.所以马上需要显示在页面的函数,不要使用lazy loading

Conditional Advance Loading

  • 和Lazy loading不同Conditional Advance Loading能够保证每次调用的时间一致,但 是代价就是所有"可能用到的function"的我们都要为其'设置变量'一次.如果这个函 数压根就没有使用到,相比于lazy loading再效率上,就有点损失. 
    var addHandler = document.body.addEventListener ?
    function(target, eventType, handler){
      target.addEventListener(eventType, handler, false); }:
    function(target, eventType, handler){
      target.attachEvent("on" + eventType, handler); };
var removeHandler = document.body.removeEventListener ?
    function(target, eventType, handler){
      target.removeEventListener(eventType, handler, false); }:
    function(target, eventType, handler){
      target.detachEvent("on" + eventType, handler); };f

Use the Fast Parts

  • 除了避短,我们有时候还可能需要"扬长". js因为依赖底层的engine,所以很多时候js 的书写者会抱怨engine的效率.但真实的情况是engine总是最快的部分,而你的代码则是 木桶中最短的一块
  • engine的实现决定了某些feature会比另外的feature快,比如如下的feature

Bitwise Operators

  • bitwise可能是最被误解的js特性,但是确实是最快的
  • 我们知道js里面的number都是被存储成64-bit format的,但是一旦两个number做了bitwise 操作,number就会被"降为"成signed 32-bit
  • 虽然存在着"降低"精度,但是这个过程却惊人的快

Native Methods

  • 无论你的代码多么的快,都不可能快过JavasScript engine提供的内置的method,原因 很简单:你的代码是js写的,而native method则是使用c++写的
  • Math.xxx就是典型的native method,我们要尽量的使用Math.xxx而不是自己去做值的 计算
  • 另外一个例子是SelectorAPI,CSS查询是被JS native实现的. 即便是被认为速度最快 的jQuery的css查询引擎,也是比native code慢的.所以在DOM操作的时候,我们要尽可 能的使用querySelector()等内置函数
