select

  • select函数的作用总结起来就是:从一个数组里面选择某些合适的成员返回一个新的函数 
    a = [1, 2, 3, 4, 5, 6, 7, 8, 9]
p a.select {|n| n % 3 == 0}

##################################################
# <===================OUTPUT===================> #
# [3, 6, 9]                                      #
##################################################

reject

  • reject函数和select函数"作用类似, 行为相反",从一个数组里面选择某些"不合适"的 成员返回一个新的函数 
    a = [1, 2, 3, 4]
p a.reject {|n| n > 2}

##################################################
# <===================OUTPUT===================> #
# [1, 2]                                         #
##################################################

Array.shift

  • Array的shift在不加参数的时候,返回第一个成员(如果一个成员都没有,就返回nil),并 且从原来的数组里面删除一个数 
    args = [ "-m", "-q", "filename" ]
p args.shift
p args

##################################################
# <===================OUTPUT===================> #
# "-m"                                           #
# ["-q", "filename"]                             #
##################################################
  • 加上参数__n__的话,就是"至少"截取n个成员 
    args = [ "-m", "-q", "filename" ]
p args.shift(2)
p args

##################################################
# <===================OUTPUT===================> #
# ["-m", "-q"]                                   #
# ["filename"]                                   #
##################################################

block 作为参赛传入函数

  • block不仅仅是看起来酷那么简单, 它在很多情况下都非常好用.比如下面
  • block作为参数传入函数, 但是你无法直接使用block, 而是: 
    block在函数内部产生"不同"作用(因为block也是函数,也可以有参数!!!), 然后产生返回值给函数用.
  • 例子如下 
    # We have num students, we want to know their test score
# test score is determined by block
def fun_with_block(num, options = {}, &block)
  score = []
  num.times do |i|
    score.push(block.call(i))
  end
  score
end

# find the first 10th students' score
s = fun_with_block(10) do |nth|
  nth * nth
end

p s

##################################################
# <===================OUTPUT===================> #
# [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]           #
##################################################

rails escape 字符

  • "<" 和 ">" 字符会引入xss攻击,所以, 在rails2的时候,使用html_escape函数把带有"<>" 的字符串转换成"&lt&gt" 
    irb(main):019:0> address_array.join('<br />')
=> "name<br />addr1<br />addr2<br />city_zip<br />country_name"
# escape the '<>'
irb(main):020:0> ERB::Util.html_escape address_array.join('<br />')
=> "name&lt;br /&gt;addr1&lt;br /&gt;addr2&lt;br /&gt;city_zip&lt;br /&gt;country_name"
  • 在模板里面其实就是使用函数h来替代html_escape 
    <%= h some_string %>
  • 在Rails 3里面,所有模板里面的字符都会自动被escape掉,所以 
    <%= h some_string %>
# is now
<%= some_string %>
  • 如果你想要"真的html效果(比如</br>就是换行,而不是显示出来)",那么你需要 raw(或者html_safe) 
    <%= raw some_string %>
# is now
<%= some_string.html_safe %>
  • raw其实是首先把some_string转换成object, 然后object调用html_safe. 所以单独使用 html_safe的时候"一定不要在一个nil对象上使用"

taint属性

  • 前面的html_safe需要作用于一个非nil的对象上面, 而taint这个属性,可以"部分的说 明某个object是不是nil"
  • 先来看看taint, 如果是用户输入的数据,那么就是tainted的 
    string = gets
string.tainted?
# => true
  • 否则,数据就是untained.但是你可以自己taint他们 
    string = 'Not yet tainted.'
string.tainted?
# => false

(string = 'Explicitly taint me!').taint
string.tainted?
# => true
  • 那么taint到底是什么意思呢? 
    Generally, you would untaint an object only after you validate and/or sanitize it.
Untainting an object marks it as "safe" for certain operations that you wouldn't
want to run on untrusted strings or other objects.
  • 简言之就是:用户输入的是不安全的(tainted),你辨别过后认为是安全的就标记其为"untainted" 
    (string = "Let's taint this string!").taint
string.untaint.tainted?
# => false
