shell的分类

  • shell其实分成了四类:
    • interactive & login shell: 登录系统(或者ssh到remote)的时候,其特点是需要输 入你的username(连username都不知道,才叫登录么). 他们会读取的配置文件代表为 .bash_profile.
    • non-interactive & login shell: 不常见
    • interactive & non-login shell: 已经登录了, 需要开启一个terminal来处理任务. 常见的有, screen, X teriminal和emacs的shell, 其会读取的配置文件代表是.bashrc
    • non-interactive & non-login shell: 当在command line里面执行script或者程序 的时候,就是调用的这种shell

不止一个配置文件

  • .bash_profile是为bash准备的.还有.profile可以为bash或者sh服务.在.bash_profile 和.profile都在的情况下,在bash里面, 是.bash_profile覆盖.profile(这也是我们主 要遇到的情况)

Mac OS X shell是个例外

  • OS X的Terminal.app和其他Linux 里面的terminal不一样, 它每次都是一个interactive &login 的shell. 所以其读取的配置文件是.bash_profile. 而不是像其他Linux的GUI terminal或者Emacs的shell一样读取.bashrc

Mac Emacs用户的推荐配置

  • 因为再Mac下面Terminal.app和Emacs的shell是读取不一样的配置文件.这给我们带来了 困扰.
  • 同样的,即便是在Linux发行版上,我们有时候也希望"interactive & login shell"和"interactive & non-login shell"是使用同样的配置文件.
  • 我们的推荐做法是,让.bash_profile读取.bashrc, 然后,配置都写在.bashrc里面 
    if [ -f ~/.bashrc ]; then
    source ~/.bashrc
fi

递归删除目录下所有指定类型(扩展名)的文件

  • 使用find找出,然后使用xargs将结果作为参数传递给rm -rf.不使用-exec是因为-exec 是一次性传递find结果,如果中间出现了"参数错误", 没有权限等,就很麻烦. xargs是 每次传递一个参数. 
    bash-3.2$ find a -name '*.txt' -type f -print
a/aa.txt
a/b/bb.txt
a/b/c/cc.txt
bash-3.2$ find a -name '*.txt' | xargs rm -rf
bash-3.2$ find a
a
a/b
a/b/c

如何替换一个文件中的某些字符串为新的字符串

  • 使用sed, 解法和vim的替换很像. 需要注意的是在mac上面参数是-ig, 而linux上面是-i 
    bash-3.2$ cat a.txt
hello old-word, I am old-word too
bash-3.2$ sed -ig 's/old-word/new-word/g' a.txt
bash-3.2$ cat a.txt
hello new-word, I am new-word too

查看当前文件夹所有内容占用硬盘的情况

  • 常见错误方法如下:使用du(意为disk usage). -h为human可读, -d为深度,为1就是当前 文件夹下. 
    bash-3.2$ du -h -d 1
1000K   ./.git
152K    ./algo
1.3M    ./notes
8.0K    ./static
2.5M    .
  • 如果你认真看就会发现上面的例子只列出了文件夹,所有当前文件夹下的文件都被忽略了. 正确的方法如下. 使用*,并且depth设置为0 
    bash-3.2$ du -h -d 0 *
520K    __MACOSX
2.5G    ct
627M    ct.zip
2.5M    org

一行在bash命令行里面进行循环

  • 可以用如下bash script在命令行打出循环 
    $ for ((i=1;i<=5;i++)); do echo NO.${i}; done

NO.3
NO.4
NO.5
  • 当然了可以在emacs里面利用这种一行的shell命令,通过C-u M-u shell-command

Linux里面的Group分成primary group和secondary group

  • 在Linux里面,早起一个用户只能属于一个group,就是primary group. 这也就是为什么 文件里面会列两行,一个user id, 一个group id.因为一个用户只能属于一个group.比如 
    hfeng@vagrant:~$ ls -al
total 24
drwxr-xr-x 2 hfeng hfeng 4096 Jul 13 09:39 .
drwxr-xr-x 4 root  root  4096 Jul  4 07:24 ..
-rw------- 1 hfeng hfeng  533 Jul 13 09:36 .bash_history
-rw-r--r-- 1 hfeng hfeng  220 Oct 29  2014 .bash_logout
-rw-r--r-- 1 hfeng hfeng 3760 Oct 29  2014 .bashrc
-rw-r--r-- 1 hfeng hfeng  675 Oct 29  2014 .profile
  • 我们使用usermod这个命令加上-g是可以更改primary group的, 而且更改以后文件的 group id也一同更改(而不是继续保留着是hfeng!) 
    hfeng@vagrant:~$ sudo usermod -g vagrant hfeng
hfeng@vagrant:~$ ls -al
total 24
drwxr-xr-x 2 hfeng vagrant 4096 Jul 13 09:39 .
drwxr-xr-x 4 root  root    4096 Jul  4 07:24 ..
-rw------- 1 hfeng vagrant  555 Jul 13 09:39 .bash_history
-rw-r--r-- 1 hfeng vagrant  220 Oct 29  2014 .bash_logout
-rw-r--r-- 1 hfeng vagrant 3760 Oct 29  2014 .bashrc
-rw-r--r-- 1 hfeng vagrant  675 Oct 29  2014 .profile
  • 如果要更改secondary的group,那么是使用-G,需要注意的是,-G的默认行为是'重置'所有 的secondary group.必须加-a才能真正成为"添加一个secondary group" 
    vagrant@vagrant:~$ groups hfeng
hfeng : vagrant docker
vagrant@vagrant:~$ sudo usermod -G cdrom hfeng
vagrant@vagrant:~$ groups hfeng
hfeng : vagrant cdrom
vagrant@vagrant:~$ sudo usermod -G docker hfeng
vagrant@vagrant:~$ groups hfeng
hfeng : vagrant docker
vagrant@vagrant:~$ sudo usermod -aG cdrom hfeng
vagrant@vagrant:~$ groups hfeng
hfeng : vagrant cdrom docker
