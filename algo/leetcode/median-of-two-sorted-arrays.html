Leetcode 002: Median of Two Sorted Arrays

题目

  • 给出两组数组，它们都已经分别排序好了，求这两个数组所有数字的中位数 
Sample Input : {1, 3, 5, 6}, {2, 4, 7, 8}
Sample Output: 4.5
Sample Input : {1, 3, 5, 7, 9}, {2, 4, 6, 8}
Sample Output: 5

思路

  • 如果不考虑复杂度,那么当然是把两个数组都放在一起,然后排序,就找到了第K个大的. 但是这种做法有很多的冗余计算:
    • 把他们拷贝到一个新的数组里面是非常浪费空间的
    • 我们不需要知道前后的大小,我只需要知道第K个
  • 所以解决这个问题就从两个地方来开启:
    • 不要额外的空间
    • 不要排序
  • 两个数组其实都已经排序了,那么我们可以安装空间把数据分成四个部分, 这其中最下的, 或者最大的一个部分都可以每次删除一个,最后就能找到第K个 
            +---------+---------+
Array A | part1   | part2   |
        +---------+---------+
        +---------------+---------------+
Array B | part3         | part4         |
        +---------------+---------------+
  • 本以为上面是最牛逼的解法了,其实还有更牛逼的解法2.上面分成四块的思想来源于K是中位 数(也就是sum/2),那么两个数组平分的话,最大的一块也不会大于sum/2(只会无限逼近 于). 就是每次都倾向于把某个子数组删除完(其中A是数组A的长度): 
    int partA = Math.min(k/2, A.length);
  • 当然解法2更牛逼的地方是,通过开始的时候让A数组始终大于B数组(如果不是就调换)的方法 来解决非常头疼的边界问题! 
    if (A.length > B.length) {
    return findNth(B, A, k);
}
  • 最后来说说总体的思路, 找到两个数组的中位数,如果总数是基数个,那么就是中间那个, 如果 总数是偶数个,那么是中间两个数的和(记得int to float). 所以这个问题转换为查找两 个排序数组的第K大的数

解答

public class Solution {
    private int findNth(int A[], int B[], int k){
        if (A.length > B.length) {
            return findNth(B, A, k);
        }
        if (A.length == 0) {
            return B[k - 1];
        }
        if (B.length == 0) {
            return A[k - 1];
        }
        if (k == 1) {
            return Math.min(A[0], B[0]);
        }
        int partA = Math.min(k/2, A.length);
        int partB = k - partA;

        if (A[partA - 1] < B[partB - 1]) {
            return findNth(Arrays.copyOfRange(A, partA, A.length), B, k - partA);
        } else {
            return findNth(A, Arrays.copyOfRange(B, partB, B.length), k - partB);
        }
    }

    public double findMedianSortedArrays(int A[], int B[]) {
        int lenSum = A.length + B.length;
        if (lenSum % 2 == 1) {
            return findNth(A, B, lenSum / 2 + 1);
        } else {
            return (findNth(A, B, lenSum / 2) + findNth(A, B, lenSum / 2 + 1)) / 2.0;
        }
    }
}
