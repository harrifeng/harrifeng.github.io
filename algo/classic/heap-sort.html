原理

  • 堆其实是一个完全二叉树(不是满二叉树,满二叉树的个数是有规定的只能是N**2 - 1), 其 父节点一定大于自己所有的子节点, 而两个子节点之间没有明确的大小关系
  • 由于完全二叉树非常整齐的特点,完全可以用数组来存储堆. 
        +----+----+----+----+----+----+----+----+----+----+
    | 9  | 8  | 7  | 6  | 4  | 4  | 3  | 2  | 1  | 0  |

  • 排序过程
    1. 最大堆调整（Max_Heapify）：将堆的末端子节点作调整，使得子节点永远小于父节点, 复杂度O(lgn)
    2. 创建最大堆（Build_Max_Heap）：将堆所有数据重新排序, 复杂度O(n)
    3. 堆排序（HeapSort）：移除位在第一个数据的根节点，并做最大堆调整的递归运算,复杂 度O(nlgn)

def max_heapify(A, root, end):
    left = root * 2 + 1
    right = root * 2 + 2
    max = root
    if left <= end and A[left] > A[root]:
        max = left
    if right <= end and A[right] > A[max]:
        max = right
    if max != root:
        A[max], A[root] = A[root], A[max]
        max_heapify(A, max, end)


def build_max_heap(A):
    size = len(A) - 1
    for i in reversed(range(0, size/2)):
        max_heapify(A, i, size)


def heap_sort(A):

应用

  • 常见的经典题目就是从10亿个数里面取最小的10个数(当然最大方法也一样,除了某些地 方反过来)
  • 常见的错误(或者说不太合理)的方法是: 找最小的10个数,我就建一个最小堆 然后从堆里面min10次出来, 这个方法当然可行,但是这样一次要把所有的数据读入内存
  • 最佳实现方法是:
    1. 在取最小的10个数的时候,用10亿个数的前十个建一个最大堆(不是写错了, 就是最大,同样取最大10个数的适合,建一个最小堆), 最大堆的堆顶肯定是这十个数 里面最大的一个了(也是最被嫌弃,最有可能被淘汰的一个)
    2. 然后用接下来的10亿个数,每次和这个最大堆堆顶进行对比, 比这个数还大的,就直接 抛弃, 比这个数小的话,就替代老的堆顶, 然后运行一次维护heap的操作,保持这个 最大堆的属性
    3. 最后剩下来的最大堆就说我们想要的前10个最小的.(前10个最大的反过来实现即可)

应用实现

import random
import heapsort


def top_n(lst, n):
    """ list contains very large number of input, n
    is relatively small number, for intance 100.
    Solve the question like "find top 10 min elements from billion"
    """
    nlist = lst[:n]
    build_max_heap(nlist)

    for num in lst[n:]:
        if num < nlist[0]:
            nlist[0] = num
            build_max_heap(nlist)
    return nlist


if __name__ == '__main__':
    aa = range(10**9)
    random.shuffle(aa)
    print top_n(aa, 10)
