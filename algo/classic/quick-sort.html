原理

  • 其实快速排序的难点，也就是在这个partition上面, 每次partition之后的数组总是“大致上排序”成功:也就是说 pivot之前的数字小于pivot, 而pivot之后的数字大于pivot
  • 最开始的时候， i指向0， j指向1， pivot是最后一个数字，value为2， j指向的数字，如果小于pivot就要和i+1 swap，这里j和i+1是同一个数字，后面我们会看到， i和j之间其实是维护了一个小的区域，他们大于pivot，每次会 派出i+1也就是区域里面第一个数字和j进行swap，来保护(i,j]之间的属性（大于pivot)

    Figure 1: Phase1

  • 6大于pviot就不需要swap，同时(i,j]区域也扩大了（绿色部分）

    Figure 2: Phase2

  • 1小于pivot，所以要swap，(i,j]保持不变

    Figure 3: Phase3

  • 3, 4, 5都大于2, 所以j一路向前(i,j]不断扩大

    Figure 4: Phase4

  • 最后把pivot和value[i+1]， swap， 然后返回i+1，因为这个是数组两个部分的分解点

实现

package algo.classic;

import java.util.Arrays;

public class QuickSort {
    public void quickSort(int[] array) {
        if (array.length <= 1) {
            return;
        }
        sort(array, 0, array.length - 1);
    }

    private int partition(int[] array, int beg, int end) {
        int pivot = array[end];
        int i = beg - 1;
        int j = beg;
        for (; j < end; j++) {
            if (array[j] < pivot) {
                i++;
                int tmp = array[j];
                array[j] = array[i];
                array[i] = tmp;
            }
        }
        int tmp = array[j];
        array[j] = array[i + 1];
        array[i + 1] = tmp;
        return i + 1;
    }

    private void sort(int[] array, int beg, int end) {
        if (beg < end) {
            int mid = partition(array, beg, end);
            sort(array, beg, mid - 1);
            sort(array, mid + 1, end);
        }
    }
}
